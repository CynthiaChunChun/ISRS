# **系統開發規格書 (SDS): 資安表單審核系統**

## **1\. System Overview**

* **Project Name**: 資安表單數位化審核系統 (Security Form Digitization & Workflow System)  
* **Objective**: 將現有紙本資安表單 (IS-D-022, IS-D-025, IS-D-029, IS-D-043, IS-D-045) 轉化為數位化流程，提升審核效率、確保審核軌跡透明化並落實資安合規性 。

* **User Stories**:  
  * **作為 申請人**，我想要 線上填寫各類資安申請單，以便 快速送審並追蹤目前處理進度 。

  * **作為 部門主管**，我想要 收到簽核通知並進行核章，以便 履行管理職責並控管部門需求 。

  * **作為 權責單位 (圖資處/資安小組)**，我想要 進行技術評估與缺失結案確認，以便 確保所有異動符合資安政策 。

## **2\. Technical Stack & Connection**

* **Language/Framework**: C\# / .NET 8 (Web API or Blazor)  
* **Oracle Client**:  
  * **Driver**: Oracle.ManagedDataAccess.Core (ODP.NET Managed Driver)  
  * **Oracle Instant Client**: Version 19c or 21c (針對存取 PL/SQL 與進階資料型別最佳化)  
* **Connection Config**:  
  * **Method**: Easy Connect Plus 或 TNS\_ADMIN  
  * **String Format**: (DESCRIPTION=(ADDRESS=(PROTOCOL=TCP)(HOST=db\_host)(PORT=1521))(CONNECT\_DATA=(SERVICE\_NAME=pdb\_name)))  
  * **Pooling**: Min Pool Size=10; Max Pool Size=100; Connection Lifetime=60;

以下整理出這五類資安表單中需要填寫的欄位詳細清單：

### **A. 防火牆進出規則申請表 (IS-D-022)**

此表單主要用於紀錄主機的網路存取需求 。

* **基本資訊：** 紀錄編號 、填表日期 、負責人 、有效日期 、主機功能用途 、Domain Name 、OS 版本 、防火牆 、其他說明 。  
* **必填欄位 (\*)：** 申請人 、IP 位址 、申請日期 、Port 、來源 IP 、目的 IP 。  
* **設定細節：** 主機名稱 、授權方式 、是否做 NAT 。  
* **核章紀錄：** 處理紀錄 、管理人員簽章 、部門主管簽章 。

### **B. 資訊服務申請表 (IS-D-025)**

用於處理帳號、權限及軟硬體的異動申請 。

* **申請基本資訊：** 紀錄編號 、填表日期 、申請人 、部門主管 、申請日期 、事由 。  
* **申請項目 (勾選)：** 申請帳號 、刪除帳號 、電腦軟硬體安裝或維修 、密碼重置 、檔案權限變更 、其它資訊服務支援 。  
* **執行與說明：** 補充說明 、審核結果 (同意/不同意) 、審核意見說明 、承辦人簽章 、預定完成日期 、主管簽章 。

### **C. 帳號權限清查紀錄表 (IS-D-029)**

用於定期審視系統帳號權限的正確性 。

* **清查標的：** 紀錄編號 、系統名稱 、清查日期 。  
* **清查清單欄位：** 項次 、帳號 、使用者 、權限說明 、備註 。  
* **核章：** 承辦人 、主管審核 。

### **D. 矯正處理單 (IS-D-043)**

針對資安事件或稽核缺失的追蹤處理 。

* **案發基本資訊：** 紀錄編號 、填表日期 、提出單位 、提出人員 、處理單位 。  
* **分類與來源：** 事件分類 (建議事項/觀察事項/次要缺失/主要缺失) 、事件來源 (資安/個資、內/外部稽核、資通安全事件自行提出、其他) 。  
* **分析與對策：** 問題或缺失說明 、原因分析 、處理人員 、研擬改善對策 、暫時性對策及預計完成日期 、永久性矯正措施 。  
* **審核與結案：** 審核意見與處理情形 、主管審核 、改善對策執行情形與完成日期 、資通安全稽核小組核章 、審核結果 (結案/其他) 。

### **E. 資訊系統開發申請表 (IS-D-045)**

包含開發申請、評估及後續測試上線的三個階段 。

* **一、申請與評估階段：**  
  * 編號 、申請日期 、希望上線日期 、申請單位及承辦人 (分機) 、系統名稱 。  
  * 需求種類：新增系統或功能、系統功能修改、其他 。  
  * 說明細節：目前作業情形 、規劃目標與需求功能說明 、預期效益 。  
  * 安全性評估：防護需求分級 (普/中/高) 、其他安全需求 。  
  * 核章：申請單位主管/承辦人 、會辦單位 。  
* **二、圖資處評估結果：**  
  * 評估結果 (同意/暫不/重大異動/其它) 、預估上線日期 、評估意見 、承辦人及主管核章 。  
* **三、測試與上線階段：**  
  * 測試日期 、測試結果 (符合並結案/不符合原因) 、預計上線日期 。  
  * 核章：申請單位主管/承辦人簽章 、會辦單位 、圖資處承辦人/主管簽章 。

## **4\. Functional Logic (IPO Focus)**

### **Module 1: 表單提交 (Form Submission)**

* **Module Name**: FormSubmissionService  
* **Data Action**: INSERT INTO SEC\_FORMS\_MASTER 及對應的資料表 (如 SEC\_DATA\_FIREWALL)。  
* **Process**:  
  1. 驗證必填欄位 (如 IP 位址、來源/目的 IP、Port) 。

  2. 根據目前年份產生 FORM\_ID (例如: 2024-FIREWALL-001)。  
  3. 設定 VERSION (對應 PDF 規範之 1.1 或 1.5) 。

  4. 寫入初始簽核軌跡至 SEC\_SIGN\_OFF\_LOG 。

* **Transaction**: 必須確保 MASTER 與 DETAIL 寫入在同一個 Transaction 內，失敗則 ROLLBACK。

### **Module 2: 系統開發評估與測試 (IS-D-045 Logic)**

* **Module Name**: SystemDevWorkflow  
* **PL/SQL Requirements**:  
  * PROC\_EVALUATE\_DEV: 傳入 FORM\_ID, RESULT (同意/暫不開發) 。

  * PROC\_TEST\_FINALIZE: 傳入測試結果、預計上線日期 。

* **Process (Pseudo-code)**:  
  C\#  
  if (currentStage \== "TESTING") {  
      UPDATE SEC\_DATA\_SYS\_DEVELOP   
      SET TEST\_RESULT \= :res, TEST\_DATE \= SYSDATE   
      WHERE FORM\_ID \= :fid;

      if (res \== "符合需求") {  
          MoveToStage("GO\_LIVE\_PENDING"); // 進入結案申請   
      }  
  }

## **5\. Security & Compliance (Enterprise Grade)**

* **Auth**:  
  * 結合 Oracle Schema 權限，應用程式帳號僅具備 EXECUTE 預存程序的權限，不直接對表進行 DML。  
* **Security**:  
  * **Bind Variables**: 強制要求所有 SQL 語法使用參數化查詢。

  C\#  
    // 正確範例 (C\#)  
    cmd.CommandText \= "SELECT \* FROM SEC\_FORMS\_MASTER WHERE FORM\_ID \= :id";  
    cmd.Parameters.Add(new OracleParameter("id", formId));

  * **Data Integrity**: 針對 CONFIDENTIALITY (機密等級) 為「敏感」之表單，CLOB 欄位可評估使用 DBMS\_CRYPTO 進行加密儲存 。

* **Constraint**:  
  * 使用 SELECT FOR UPDATE WAIT 5 處理並發簽核，防止長事務導致的 Oracle 鎖表 (Deadlock)。

### **Code-Ready Notes**

* **SYSDATE**: 統一使用 Oracle 伺服器時間，避免 Web Server 與 DB Server 時區不一致。  
* **NVL Handling**: 在查詢簽核意見時，使用 NVL(COMMENTS, '無意見') 處理 CLOB 為空的情況 。

* **Identity Result**: 使用 RETURNING FORM\_ID INTO :id 語法在 INSERT 後立即取得產生的識別碼。
